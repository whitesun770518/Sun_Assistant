# 나만의 비서실 에이전트 개별 MD 파일 묶음

이 폴더는 비서실형 멀티 에이전트 운영을 위해 **개별 에이전트별 마크다운 파일**로 분리한 구성임.

## 운영 방식 핵심
- 각 에이전트가 먼저 자신의 전문 의견을 냄
- 에이전트 간 토의를 통해 강점·한계·누락 쟁점을 검토함
- **서기**가 토의 과정을 기록함
- 최종 결론은 단일 주장만이 아니라 **전문가별 의견과 쟁점, 보류사항을 포함**해 정리함
- 모든 파일에 업데이트 로그, 최근 사용 메모, 토의 보완 메모, 다음 개선 과제를 넣어 지속적으로 보완할 수 있게 구성함

## 추천 사용 순서
1. 현안 입력
2. 관련 에이전트별 1차 의견 생성
3. 서기가 토의 기록
4. 비평가·팩트체커·법률가가 검증
5. 보좌관이 최종 실무 문안화
6. 프로젝트 매니저가 후속 일정과 담당 정리

## 추천 폴더 운영 팁
- 개별 파일은 계속 수정하고, 별도로 `회의기록` 폴더를 만들어 서기 문서를 쌓아두면 관리가 쉬움
- 실제로 자주 쓰는 프롬프트는 각 파일 하단의 `빠른 호출용 프롬프트 초안`을 수정하며 고정해두면 됨
- 에이전트 역할 변경 시 반드시 `업데이트 로그`에 이유와 효과를 함께 남기는 것을 권장함
